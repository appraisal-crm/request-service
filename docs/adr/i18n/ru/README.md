# Architecture Decision Records

> Русская версия · [English version](../../README.md)

ADR документирует значимое архитектурное решение: что решили, почему и каковы компромиссы.

## Индекс

| ADR | Название | Статус |
|-----|----------|--------|
| [ADR-001](ADR-001-kafka.md) | Apache Kafka в качестве брокера сообщений | На рассмотрении |
| [ADR-002](ADR-002-keycloak.md) | Keycloak для аутентификации и авторизации | На рассмотрении |
| [ADR-003](ADR-003-database-per-service.md) | База данных на сервис | На рассмотрении |
| [ADR-004](ADR-004-open-source-license.md) | Открытая лицензия (MIT / Apache 2.0) | На рассмотрении |
| [ADR-005](ADR-005-go-backend.md) | Go для бэкенд-сервисов | На рассмотрении |
| [ADR-006](ADR-006-s3-storage.md) | S3 Yandex Cloud для хранения файлов | На рассмотрении |
| [ADR-007](ADR-007-event-delivery-outbox.md) | Надёжная публикация событий через Transactional Outbox | На рассмотрении |

## Статусы

- **Proposed** — обсуждается
- **Accepted** — решение принято и действует
- **Deprecated** — больше не актуально
- **Superseded** — заменено более новым ADR
